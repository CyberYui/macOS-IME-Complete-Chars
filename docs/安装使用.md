# 安装使用指南
本指南详细说明plist文件的导入步骤、测试方法及常见问题解决办法，确保顺利使用。

## 一、选择导入方案

| 方案 | 文件 | 条数 | 适合人群 |
|------|------|------|----------|
| 精简版（推荐） | `cannot-type.plist` | 39条 | 只想补全真正打不出的字符，零干扰 |
| 无冲突版 | `no-conflict.plist` | 70条 | 日常输入体验优先，不希望生僻字干扰常用字 |
| 完整版 | `radicals.plist` + `rare-chars.plist` | 111条 | 需要完整字符覆盖，可接受偶尔干扰 |

## 二、导入步骤（必看）
1.  从GitHub仓库下载plist文件：进入仓库 → plist目录 → 下载 `radicals.plist`（偏旁部首）、`rare-chars.plist`（生僻字），或直接下载 `no-conflict.plist`
2.  打开Mac系统设置：点击屏幕左上角苹果图标 → 系统设置 → 找到「键盘」选项
3.  进入自定义短语：在键盘设置中，点击「文本输入」→「编辑...」→ 找到「自定义短语」栏目
4.  导入文件：将下载的plist文件，直接拖入「自定义短语」的列表中，系统会自动识别并导入
5.  验证导入：导入后，列表中会显示所有字符及对应拼音，说明导入成功

## 二、测试方法
1.  切换输入法：将Mac输入法切换为「简体拼音」（原生）
2.  测试触发：输入字符对应的拼音，目标字符会优先出现在候选列表前排（如输入ba，可优先打出丷；输入pu，可打出攴）
3.  异常排查：若无法触发，可先对照 `CONTENTS.md` 确认触发拼音是否正确；仍有问题可询问AI：

> 「我在macOS原生简体拼音输入法中导入了自定义短语plist文件，输入「xxx」拼音后候选列表中没有出现「X」字符，可能是什么原因？怎么排查？」

## 三、常见问题

### 1. plist文件无法导入
- 重新从GitHub下载，确保文件未被修改，直接拖入不要用编辑器打开
- 若仍失败，可将plist文件内容粘贴给AI询问：「这个plist文件格式是否符合macOS自定义短语的要求？」

### 2. 输入拼音无法触发字符
- 确认输入法为原生简体拼音（非搜狗、百度等第三方）
- 对照 `CONTENTS.md` 核对触发拼音
- 确认「自定义短语」功能已启用（系统设置→键盘→文本输入→编辑...）

### 3. 导入后字符排名未提升
- 多使用几次目标字符，输入法会逐渐学习提升排名
- 也可询问AI：「macOS原生输入法自定义短语导入后，如何让候选排名更靠前？」

### 4. 导入后字符重复
- 删除自定义短语列表中的重复条目，重新导入本仓库plist文件
- 可询问AI：「如何批量清理macOS自定义短语中的重复条目？」

## 四、注意事项
1.  本plist文件仅适用于macOS原生简体拼音，第三方输入法（如搜狗、百度）无法使用
2.  导入后无需额外设置，直接输入拼音即可触发，无需添加其他前缀/后缀
3.  若系统更新后，部分字符无法触发，可重新导入plist文件
